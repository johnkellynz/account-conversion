Why this document exists

DE's HVAC Victaulic Key Account Conversion Strategy — Australia sets the corporate playbook. This Q&A captures the 50 design decisions needed to turn the existing Field CRM into the operational layer of that strategy: the system reps run their day out of, and the system management pulls KPIs from.

Each question was answered by JK. Default recommendations are marked. Two answers diverged from the recommendation (Q19, Q26) and one was custom (Q26 product taxonomy) — flagged below for DE's view. The "Open Tensions" section at the end lists items to resolve before build kicks off.

1. Goals & Users (Q1–Q5)

Q112 months from now, what does "winning" look like for this system?
AnswerPipeline + KPI engine for management — the system generates the PSA / CIP / KAC-revenue KPIs from strategy section 10.1, ready for management each quarter.
Q2Who logs in and uses it?
AnswerFull ANZ field team — multiple reps across states, with patch assignment, account ownership and sharing rules.
Q3Sign-in method preference?
Answer (recommended)Email + password + 2FA. Standard, hand-over-friendly. 2FA via authenticator app blocks credential stuffing.
Q4Where should the data physically live?
Answer (custom)"Supabase will be fine so long as it's secure." — Sydney region; Victaulic IT sign-off optional but recommended for sensitive contractor data.
Q5With a full ANZ field team, how should roles work?
Answer (recommended)Rep / Manager / Admin. Reps enter & see their patch. Managers see all data + dashboards. Admin manages users. Clean separation.

2. Sharing, Devices & Account Model (Q6–Q9)

Q6Should reps see each other's accounts?
Answer (recommended)Yes — fully open across reps. Best for cross-state contractors & team coverage. Audit log shows who edited what.
Q7What is one "account" in the system?
Answer (recommended)One per company, with state branches linked (e.g. Watpac master → Watpac SYD / BNE / MEL sub-offices). Supports roll-up + per-state visibility.
Q8Primary device for daily use?
Answer (recommended)iPad on site + laptop at desk. iPad-first interface for client visits; full desktop view for planning; phone is quick-note.
Q9Where does the conversion stage (IDENTIFY → ENGAGE → DEMO → CONVERT → GROW) sit?
Answer (recommended)One stage per account, plus per-product status. Account stage drives pipeline view; per-product status drives cross-sell. Both reportable.

3. PSA, Projects & Displacement (Q10–Q14)

Q10How should the PSA tier (None / Silver / Gold / Platinum) be tracked?
Answer (recommended)Current tier + target tier + renewal date. Drives QBR planning.
Q11How should "project" be modelled?
Answer (recommended)Pipeline stages: Lead → Tender → Won → In-build → Complete. Tracks competing contractors per project. Forecast Vic content $ × probability.
Q12For tracking incumbent piping systems on each contractor?
Answer (recommended)Per contractor — default piping spec + notes. Drives section 6.1 / 6.2 / 6.3 displacement playbooks.
Q13How should buyer personas be tagged on contacts?
Answer (recommended)One primary persona + optional secondary (Estimating Mgr / Project Eng / Procurement / BIM Lead / Site Foreman / MD-Director). Drives section 5 value-message templates.
Q14How important is offline data entry on iPad / phone?
Answer (recommended)Critical — site visits often have no signal. Full offline read + write; sync on reconnect.

4. Photos, Dashboard & Auth Detail (Q15–Q18)

Q15Photo capture on interactions / projects / contacts?
Answer (recommended)Yes — stored in Supabase, attached to records (business cards, site signage, project conditions).
Q16What's the Monday-morning dashboard?
Answer (recommended)My week — overdue actions + this week's meetings + tender deadlines. Personal action list first.
Q172FA method preference?
Answer (recommended)Authenticator app (Google / Microsoft / Authy). Most secure, works offline, standard for Supabase MFA.
Q18How often should the app force re-login?
Answer (recommended)Stay logged in until manual logout; re-verify 2FA every 30 days. Convenient for daily use, periodic re-check.

5. Security, Encryption & Privacy (Q19–Q22)

Q19Lost-device / leaving-employee kill switch?
Answer (recommended)Admin can force-logout any user from any device. Critical for stolen iPads & departing staff.
Q20Field-level encryption for sensitive notes?
Answer (recommended baseline)Standard Supabase encryption at rest + TLS in transit. Strong default; no client-side / no per-field extra encryption.
Q21What's the worst-case data leak you most want to prevent?
AnswerAll three equally: competitor seeing the hit list, contractor reading own internal notes, public PII leak. Drives strict access controls across the board.
Q22Audit log — who changed what, when?
Answer (recommended)Full audit log on every record change. Visible to admin. Essential for multi-user trust.

6. Compliance, Backups & Wallet (Q23–Q26)

Q23Australian Privacy Act compliance — right to be deleted?
Answer (recommended)Yes — contacts can be deleted on request, with logged action. Wipes PII, preserves company / project record.
Q24Backups & recovery?
Answer (recommended)Daily automated backup + 30-day restore window (Supabase Pro tier). Point-in-time restore.
Q25Share-of-wallet tracking format?
Answer (recommended)Annual snapshot per company — Vic revenue, est. total spend, share %. Matches strategy's 15–20% → 40–50% target trajectory.
Q26What Vic product taxonomy do you want to track per account?
Answer (custom)Mechanical (HVAC), Water, Infrastructure, Mining. Note: this is a sector-based taxonomy (matches existing CRM MARKETS array) rather than the strategy's product-line taxonomy (Grooved / Valves / Fire / Factory Solutions / Vortex). See Open Tension #1.

7. Bid Support, CIP, KPIs & QBR (Q27–Q30)

Q27Tender / bid support tracking?
Answer (recommended)Tag projects needing bid support + deadline + status. Dashboard view for engineering-support workload. Mirrors strategy section 7.1 Bid Support Service.
Q28CIP (Certified Installer Program) tracking?
Answer (recommended)Per individual — status + date + cert expiry (Not invited / Invited / Enrolled / Certified / Expired). Supports the 40 → 90 → 160 individual target.
Q29What KPI report does management want first?
Answer (recommended)Monthly pipeline pack — accounts by stage, $ forecast, PSAs signed. Auto-generated. Mirrors strategy section 10.1 KPI table.
Q30Can the app auto-generate a QBR pack per account?
Answer (recommended)Yes — one-click PDF / PPTX per account, rendered from current data. Account Plan template (strategy section 9) populated automatically.

8. Hit List, Handover & Notifications (Q31–Q34)

Q31How should the Tier 1 hit list be seeded?
Answer (recommended)Pre-seed all 8 nationals (Watpac, JCI/York, Stowe, Hastie/Ventia, Mechanical One, Nilsen, AE Smith, Intech) with conversion status from the strategy doc. Ready for day one.
Q32Handover plan if you leave Victaulic?
Answer (recommended)Built-in admin transfer flow. Successor gets admin rights without your password. Audit log preserved.
Q33Notifications & alerts — how should the app reach you?
Answer (recommended)Push to phone / iPad + 7am daily email digest. Real-time for urgent (tender deadline tomorrow), summary for daily planning.
Q34Voice-to-text note capture on phone?
Answer (recommended)Yes — iOS native dictation for in-car post-meeting notes. Huge field-rep time saver.

9. Maps, Integrations & App Format (Q35–Q38)

Q35Maps & geocoding — how visual?
Answer (recommended)Current map view + filter by stage / PSA / sector. Build on what's there. Plan a state visit visually.
Q36Integrations — what should the CRM talk to?
Answer (all four)Outlook calendar (meetings auto-logged), Outlook email (sent emails attached to contact), LinkedIn (connected status / profile sync), Victaulic SAP / global CRM (actuals in, opportunities out). Note: SAP and Outlook email require Victaulic IT engagement and live in Phase 3.
Q37Native iOS app vs web app for iPad / phone?
Answer (recommended)Progressive Web App (PWA) — install icon, works offline, no App Store. Fast iteration, built into current stack.
Q38Lunch & Learn tracking depth?
Answer (recommended)Keep current schema (company, date, topics, attendees, notes). Working as-is; revisit if reporting needs grow.

10. References, Competitor, UX & Risk (Q39–Q42)

Q39Reference project network (peer-to-peer references)?
Answer (recommended)Flag projects as "willing to be a reference" + contact + sectors. When pitching, app suggests a peer call. Strategy section 7.1.
Q40Competitor / displacement tracking on projects?
Answer (recommended)Per project — incumbent spec + who we're displacing + status. Drives section 6 playbook per job, not just per account.
Q41How should the Account Plan screen feel?
Answer (recommended)Single scrollable page with collapsible sections (Overview / Stakeholders / Pipeline / Spend / Actions / Risks). iPad-friendly.
Q42Risk register — do you want it per account?
Answer (recommended)Yes — per-account risks + mitigation + RAG status. Surfaces "Red" accounts on dashboard. Strategy section 9 Account Plan template.

11. Forecast, Pricing, Migration & Patches (Q43–Q46)

Q43Pipeline forecast — how should it be calculated?
Answer (recommended)Per project — Vic content $ × probability %, auto-rollup to account / state / quarter. Standard sales-forecast model.
Q44Should sales pricing or PSA discount % be stored in the CRM?
Answer (recommended)Yes — PSA discount tier + project pricing notes stored, with restricted access. Critical for QBR pack.
Q45Your existing CRM data — what should happen to it?
Answer (recommended)Migrate all existing customers / interactions / projects into the new schema. Backfill new fields blank. Zero data loss.
Q46How should reps be assigned to accounts?
Answer (recommended)Geographic state patches (NSW / VIC / QLD / WA / SA). National Tier 1 accounts co-owned across states. Matches strategy's 5-city focus.

12. Events, Build Approach, Notes & Rollout (Q47–Q50)

Q47Track industry event attendance (AMCA, ARBS, Victaulic Summits)?
Answer (recommended)Yes — events calendar + attendance log + leads captured. Tag interactions to an event. Strategy section 7.2 calendar built in.
Q48Build approach & timeline appetite?
Answer (recommended)Phased — ship foundations in 4 weeks, add features monthly. Phase 1: schema + auth + roles + Account Plan view. Phase 2: forecast + KPI pack. Phase 3: integrations + QBR PDF + voice.
Q49"Sensitive notes" field — who can see?
AnswerPrivate notes opt-in — default open across team; mark a note as "private" and only owner + admin can see it. Best for sensitive items like PSA negotiations.
Q50Launch plan for the ANZ team?
Answer (recommended)Pilot with JK for 4 weeks, then 2 more reps, then full team. De-risk solo, onboard in waves with training.

Open Tensions — for DE's view before build kicks off

Items needing alignment before Phase 1 begins

  1. Product taxonomy split. JK selected a sector-based taxonomy (HVAC / Water / Infrastructure / Mining) for per-account product tracking. The strategy's KPIs in section 10.1 are product-line-based (Grooved / Valves / Fire / Factory Solutions / Vortex). The monthly pipeline pack may need to report both. Decision needed: which view drives the KPI pack?
  2. PSA tier vs. share of wallet. Strategy section 8 ties PSA tier to AUD revenue bands (Silver $150K–500K, Gold $500K–1.5M, Platinum $1.5M+). Should the tier be auto-derived from the share-of-wallet snapshot, or manually set by the account owner and audited at QBR? Decision affects the schema design.
  3. National account co-ownership rules. With state patches + co-owned Tier 1 nationals, the UI needs clear rules: who is lead rep for Watpac (across SYD / BNE / MEL)? Who can edit the master account record vs. state-branch records? Operating rule needed.
  4. Outlook email integration scope. Calendar sync is straightforward; sent-email sync needs Microsoft Graph permissions from Victaulic IT and is substantially more work. Likely Phase 3. Confirm priority.
  5. SAP integration uncertainty. Pulling actuals from SAP depends entirely on Victaulic IT willingness and project bandwidth. Plan must work without it for at least Phases 1 and 2. Need a sponsor on the Vic IT side.
  6. Victaulic IT sign-off on Supabase. Data residency is Supabase Sydney (acceptable to JK). Recommend looping Vic IT in for review before Phase 1 build, given the data sensitivity. Who from IT is the right contact?
